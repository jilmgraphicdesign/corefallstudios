What is this?

This is a pretend (safe) command line terminal. You type simple words to look around and open files.

How to unlock the terminal

  1. Must be logged into the CSO Admin account to access terminal.
  2. With CSO Admin access, visit the three Intel pages on the Employee Intranet.
  3. Each page shows a Base64 string. Decode them to get three words.
  4. On the Admin page, enter the phrase (three words together) to unlock.

Your code checks for the phrase can use upper/lowercase, doesn’t matter.

Quick commands
help               show the list of commands
ls                 list what's here
cd  'folder name'  move into a folder (example: cd /logs)
cd                 exit the folder you are in and reset
cat 'file name'    open a file (example: cat alerts.log)
run 'program name' run a tool (example: run disable_safe_mode)
oog data logs      show out-of-game incident notes
clear              wipe the screen, (does not remove you from the file you are in)
hint               a small nudge
Files              list all file names
Folders            list all folder names

⚠️ SYSTEM NOTES

Some systems appear intentionally obscured. Logs suggest external interference in /utopia.

Step-by-step examples

  1. Start at the root — type: ls
    You should see folders including logs and secrets.
  2. Inspect system logs first — type: cd /logs
  3. Read the alerts for hints — type: cat alerts.log
    Look for references to hidden system structure or secure data.
  4. Return to root — type: cd /
  5. Enter the secure directory — type: cd /secrets
    This area contains protected system intelligence.
  6. List hidden files — type: ls
    You should now see encrypted system components.
  7. Open the cipher key — type: cat cipher.key
    This file contains faction-level metadata and core encryption context.
  8. Optional exploration path — type: cat worm.map
    Provides anomaly tracking and deeper system corruption clues.

Troubleshooting

  • “No such file.” — You tried to open a folder as a file, or misspelled the name. Use ls to see exact names.
  • Absolute paths — You can always use the full path, e.g. cat /logs/attacks.feed.
  • Programs live in /bin — Use cd /bin then ls to see them.
  • Stuck? — Type help or hint.
